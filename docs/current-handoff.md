## Decisions and open questions

**Decisions made:**

- **Auth: email + password + email OTP on fresh sign-in.** Replaces passkey for providers. Reason: providers are training-provider directors / marketing managers, not necessarily tech-fluent; passkey requires Face ID / Touch ID / Windows Hello which is a wall on the first screen. Email + password is the universal B2B SaaS pattern; email-OTP as 2FA is simpler than authenticator apps for the audience. Cookie keeps signed in ~7 days, so day-to-day is open-and-go. Passkey infrastructure retired but files kept in place (provider_passkeys table still exists, just unused).
- **Demo provider stays.** `demo-provider-ltd` kept for ongoing portal testing. Its leads excluded from `/admin/actions` so test data doesn't pollute the queue. `hello@charlie-harris.com` test user kept on it for sanity-checking changes.
- **No global audit page; per-provider tab only.** Charlotte's call. Reduces cognitive load and matches the natural drill-down path (open a provider → see their activity).
- **Audit reads through `public.vw_audit_actions` view.** Even after adding `audit` to Supabase Data API exposed schemas + reloading PostgREST schema cache, `supabase-js .schema("audit")` kept throwing "Invalid schema: audit". Workaround is a `security_invoker` view in public that the underlying RLS / GRANTs still gate.
- **Course cohort filter seeded from a canonical table.** Per-lead intake fields aren't enough — leads pre-dating the fastrack form have NULL intake fields, so filter only showed dates that happened to appear in routed-lead data. New `crm.course_intakes` table is the source of truth for "currently open intakes per course"; portal unions seeded options with row-derived ones.
- **Site URL is `https://app.switchleads.co.uk`.** Auth emails default to provider host. Admin reset flows pass explicit redirect to admin host. Providers never bounce to admin subdomain.
- **Custom SMTP via Brevo for all Supabase Auth emails.** Removes Supabase footer + built-in 4-emails-per-hour rate limit. Auth email templates still live in Supabase dashboard (HTML edited there); Brevo just delivers.
- **GDPR retention soft-anonymise vs erasure hard-delete.** Retention cron NULLs PII columns on `leads.submissions` (and cascades to `crm.lead_notes.body`, `fastrack_submissions.voice_of_learner_intro`, `crm.enrolments.outcome_note`) for rows older than 24 months with no active enrolment — preserves operational metadata for analytics. Right-to-erasure tool hard-deletes everything for that learner across DB + Brevo + provider sheets. Different rights, different tools.
